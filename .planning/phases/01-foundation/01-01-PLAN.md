    1. Create `src/lib/supabase/server.ts` â€” async factory function for server-side Supabase client:
       - Import `createServerClient` and `CookieOptions` from `@supabase/ssr`
       - Import `cookies` from `next/headers`
       - Export async function `createClient()` that:
         - Gets `cookieStore` via `await cookies()`
         - Returns `createServerClient(process.env.NEXT_PUBLIC_SUPABASE_URL!, process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!, { cookies: { getAll, setAll } })`
         - `getAll` returns `cookieStore.getAll()`
         - `setAll` wraps `cookieStore.set()` calls in try/catch (Server Components are read-only, middleware will persist)
       - Follow exact pattern from RESEARCH.md Pattern 1.

    2. Create `src/lib/supabase/client.ts` â€” browser client factory:
       - Import `createBrowserClient` from `@supabase/ssr`
       - Export function `createClient()` that returns `createBrowserClient(process.env.NEXT_PUBLIC_SUPABASE_URL!, process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!)`
       - Follow exact pattern from RESEARCH.md Pattern 2.

    3. Create `src/lib/supabase/middleware.ts` â€” session refresh + route protection:
       - Import `createServerClient`, `CookieOptions` from `@supabase/ssr`
       - Import `NextResponse`, `NextRequest` from `next/server`
       - Export async function `updateSession(request: NextRequest)` that:
         - Creates `response = NextResponse.next({ request })`
         - Creates Supabase client with request/response cookie bridge (getAll from request.cookies, setAll to response.cookies)
         - Calls `const { data: { user } } = await supabase.auth.getUser()` (NEVER getSession â€” security per RESEARCH.md)
         - Defines auth routes: paths starting with `/login`, `/reset-password`, `/auth/confirm`
         - If `!user && !isAuthRoute`:
           - If path starts with `/api/`: return `NextResponse.json({ error: { code: 'UNAUTHORIZED', message: 'Authentication required' } }, { status: 401 })` (per D-06)
           - Otherwise: return `NextResponse.redirect(new URL('/login', request.url))` (per AUTH-03)
         - If `user && pathname === '/login'`: redirect to `/` (prevent authenticated users seeing login)
         - Otherwise: return response
       - Follow exact pattern from RESEARCH.md Pattern 3.

    4. Create `src/middleware.ts` â€” root middleware entry point:
       - Import `NextRequest` from `next/server`
       - Import `updateSession` from `@/lib/supabase/middleware`
       - Export async function `middleware(request: NextRequest)` that returns `updateSession(request)`
       - Export `config` with matcher: `['/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)']`
       - This matcher covers all routes except static assets.
  </action>
