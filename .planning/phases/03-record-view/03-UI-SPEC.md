## Interaction States

### Page Loading (4-state: loading)

Skeleton layout: one line for back link, one block for header, horizontal skeleton bars for tabs, card skeleton for content area. Use shadcn Skeleton component.

### Page Error (4-state: error)

Alert component with `variant="destructive"`. Copy: "Не удалось загрузить запись. Попробуйте обновить страницу." Back link still visible.

### Page Not Found

Alert component (non-destructive). Copy: "Запись не найдена." Back link visible.

### Empty Tab (D-04)

Tab always visible. Content area shows: `<p className="text-muted-foreground text-sm py-8 text-center">Ничего не найдено</p>`

### Tab Navigation

Standard shadcn Tabs behavior. Active tab has underline indicator via Tabs component default styling. All 6 tabs always visible. No icons in tab labels.

### Edit Mode (D-10 through D-13)

| State | Visible Controls | Behavior |
|-------|-----------------|----------|
| View mode | "Редактировать" button (top-right, next to tabs) | Text fields rendered as static text |
| Edit mode | "Сохранить" + "Отмена" buttons (replace "Редактировать") | Text fields become Input/Textarea, editable. Enum fields stay read-only with visual indicator (opacity-50 or disabled styling) |
| Saving | "Сохранить" button shows loading spinner, disabled | PATCH request in flight |
| Save error | Inline Alert below buttons | Copy: "Не удалось сохранить изменения. Попробуйте ещё раз." |
| Save success | Return to view mode | Updated data reflected immediately |

Edit mode applies across ALL tabs simultaneously. User can switch tabs while in edit mode. Unsaved changes persist across tab switches (held in React state). Only "Сохранить" persists to DB.

### Evidence Collapsible (D-09)

Default: collapsed. Shows claim text as normal paragraph. Below it: clickable text link "источник" with a chevron-down icon (lucide `ChevronDown`). On expand: evidence quote in italic + speaker name in muted small text. Chevron rotates to up.

## Interpretation Tab Layout (D-08)

Render blocks in this exact order:

1. **Статус** -- StatusBadge + real_status_reason as body text below
2. **Резюме** -- summary as paragraph
3. **Управленческий взгляд** -- management_view as paragraph
4. **Скрытые блокеры** -- list of EvidenceCollapsible components. Empty state if array is empty.
5. **Неясности** -- list of EvidenceCollapsible components. Empty state if array is empty.
6. **Что уточнить** -- numbered list of strings. Empty state if array is empty.

Each section separated by 24px (lg) vertical gap. Section headings use Heading typography (20px / 600).

## Extraction Tab Details

### Tab: Сделано (done[])

Each item: `description` as body text. If `person` present: show in muted small text after description.

### Tab: В работе (in_progress[])

Each item: `description` as body text. If `person` present: muted small text. If `deadline` present: muted small text with calendar icon (lucide `Calendar`, 14px).

### Tab: Блокеры (blockers[])

Each item: ImpactBadge (colored per D-05) + `description` as body text on the same line. Badge appears before description.

### Tab: Назначения (assignments[]) (D-07)

Grouped by person. `person` as label (14px / 500). Below: unordered list of `tasks[]`. If `by_when` present: muted small text below task list.

### Tab: Дедлайны (deadlines[])

Each item: `date` in label weight + `description` as body text. If `type === 'inferred'`: InferredBadge ("выведен") appears after the date. If `condition` present: muted small text below.

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| Tab keyboard navigation | Built-in via shadcn Tabs (radix) -- arrow keys between tabs, Enter/Space to activate |
| Edit mode toggle | `<button>` element, not div. Announces state change via aria-pressed or aria-expanded |
| Evidence collapsible | Built-in via shadcn Collapsible (radix) -- aria-expanded on trigger, content hidden when collapsed |
| Impact badges | Include sr-only text: "Влияние: высокий/средний/низкий" |
| StatusBadge | Already has visible text label ("В порядке" / "Есть вопросы" / "Требует внимания") |
| Back link | Use `<a>` or Next.js `<Link>` with visible text "Назад", not icon-only |
| Form inputs in edit mode | Each input has associated `<Label>` (visible or sr-only) |
| Save/Cancel focus | After save success, focus moves to "Редактировать" button. After cancel, same. |
