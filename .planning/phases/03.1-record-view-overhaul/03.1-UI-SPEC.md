## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-text gaps (e.g. calendar icon to date text), badge internal padding |
| sm | 8px | Card internal padding between stacked elements, gap between badges |
| md | 16px | Card body padding (px-4), column header to first card gap |
| lg | 24px | Gap between kanban columns, section spacing within interpretation |
| xl | 32px | Page-level horizontal padding, kanban container side margins |
| 2xl | 48px | Top margin from page header to kanban grid |
| 3xl | 64px | Not used in this phase |

Exceptions: Card size="sm" uses 12px (px-3) internal padding per existing shadcn Card component. This is an existing convention, not a new exception.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (normal) | 1.6 (leading-relaxed) | Card description text, task text, source quotes |
| Label | 12px (text-xs) | 400 (normal) | 1.4 | Column subtitles, badge text, muted metadata (person, deadline slot) |
| Heading | 16px (text-base) | 600 (semibold) | 1.375 (leading-snug) | Column titles, card person name (assignments) |
| Page title | 20px (text-xl) | 600 (semibold) | 1.2 | Record page header (already exists in record-header.tsx) |

Notes:
- Weights limited to 400 and 600 project-wide. No other weights used.
- shadcn CardTitle applies `font-semibold` by default, which aligns with the 600 weight declared here. No override needed.
- All body text uses `leading-relaxed` (1.625) for readability in dense kanban cards.

## Visual Hierarchy

Primary visual anchor: the kanban board grid occupying the main content area. Page header (title + StatusBadge) is secondary orientation context. The user's eye should land on the columns first, then scan column titles to orient, then individual cards.

## Layout Contract

### Kanban Grid

```
[Page Header: title + StatusBadge + metadata]
                    48px gap
[Kanban Board — 5 columns, horizontal scroll on < 1280px]
  | Сделано      | В работе     | Блокеры      | Назначения   | Дедлайны     |
  | subtitle     | subtitle     | subtitle     | subtitle     | subtitle     |
  | [card]       | [card]       | [card]       | [card]       | [card]       |
  | [card]       | [card]       | [blocker]    | [card]       | [card]       |
  |              |              | [AI blocker] |              |              |
                    48px gap
[Interpretation Section — full width below kanban]
```

- Kanban container: `grid grid-cols-5 gap-6` on desktop (>= 1280px).
- Below 1280px: `flex overflow-x-auto gap-6` with `min-w-[240px]` per column, horizontal scroll.
- Below 768px: single column stack with collapsible column headers (columns become accordions).
- Each column has vertical overflow: `max-h-[calc(100vh-280px)] overflow-y-auto` with subtle scrollbar.
- Interpretation section renders below kanban at full page width (not inside a column).

### Card Layout (View Mode)

```
[Card — ring-1 ring-foreground/10, rounded-xl]
  [CardContent]
    [description text — 14px, leading-relaxed]
    [person — 12px, muted, optional]
    [ImpactBadge — blockers only, optional]
    [SourceQuoteButton — if source_quote exists]
  [DeadlineSlot — conditional footer]
    [Calendar icon 14px + formatted date — 12px]
```

### Card Layout (Edit Mode — D-03, D-06)

```
[Card — ring-2 ring-primary, rounded-xl]
  [CardHeader]
    [CardAction: cancel button (X icon)]
  [CardContent]
    [Textarea — description]
    [Input — person, optional]
    [Input — deadline/by_when, optional]
    [ImpactBadge — read-only, opacity-60]
    [Alert — autosave error, conditional (D-05)]
```

- Transition: card expands smoothly via CSS `transition-all duration-200`.
- Click outside triggers blur handler that calls autosave Server Action.
- On autosave failure: error Alert appears inside card, fields remain editable, data not lost (D-05).

## Interaction Contract

### Card Click-to-Edit (D-03, D-04, D-06)

| Trigger | Action |
|---------|--------|
| Click on card (view mode) | Card expands to edit form. All text fields become editable. Ring changes to `ring-2 ring-primary`. |
| Click cancel button (X) | Revert to original data. Collapse card. No save. |
| Click outside card (edit mode) | Autosave via Server Action. On success: collapse card, show updated data. On failure: show error inside card (D-05). |
| Press Escape (edit mode) | Same as cancel button — revert and collapse. |

### Source Quote (D-10, D-11)

| Trigger | Action |
|---------|--------|
| Click "источник" on card | Collapsible expands below card description showing italic quoted text. Same pattern as existing EvidenceCollapsible. |
| No source_quote on item | Button not rendered. No empty state. |

### Hidden Blockers in Blockers Column (D-02)

| Trigger | Action |
|---------|--------|
| Render blockers column | First: explicit blockers from extracted_data. Then: hidden_blockers from interpretation, each with amber background + "AI" badge. |
| Click on AI blocker card | Expands evidence source (EvidenceCollapsible pattern). NOT editable — these come from interpretation, not extracted_data. |

### Meeting Date Picker (D-12, D-13, D-14)

| Trigger | Action |
|---------|--------|
| Page load (AnalyzeForm) | Date picker pre-filled with today's date. |
| User changes date | Updates meeting_date state. No validation required — any valid date accepted. |
| User submits without changing | Today's date sent as meeting_date (D-13). |

### Column Overflow Scroll

| Trigger | Action |
|---------|--------|
| Column content exceeds viewport | Column scrolls vertically. Scroll indicator (subtle gradient fade at bottom) appears. Column header stays fixed at top. |

## Date Formatting Contract (D-09)

| Input | Display Format | Example |
|-------|---------------|---------|
| Valid ISO datetime | `d MMMM` (Russian locale) | 2026-04-04T00:00:00+03:00 -> "4 апреля" |
| Valid ISO datetime (different year) | `d MMMM yyyy` | 2027-01-15T00:00:00+03:00 -> "15 января 2027" |
| Non-ISO text (legacy) | Raw text as-is | "до конца недели" -> "до конца недели" |
| null / undefined | Slot not rendered | (no deadline slot) |

Use `Intl.DateTimeFormat('ru', { day: 'numeric', month: 'long' })` for formatting. No external date library required.

## Accessibility Contract

| Element | Requirement |
|---------|------------|

| KanbanCard (edit mode) | Focus trapped within card. Escape exits edit mode. |

## States Contract (4-State Pattern)

### Record Page

| State | Visual |
|-------|--------|
| Loading | 5 skeleton columns with 2-3 skeleton cards each. Skeleton uses existing `<Skeleton>` component. |
| Empty (no extracted data) | Full-width message: "Данные извлечения отсутствуют" centered. |
| Error (failed to load record) | `<Alert variant="destructive">`: "Не удалось загрузить запись. Вернитесь на главную и попробуйте снова." |
| Success | Kanban board renders with data. Empty columns show column-specific empty state copy (see Copywriting Contract). |

### Per-Card Autosave

| State | Visual |
|-------|--------|
| Saving | Brief opacity reduction (opacity-70) on card during save. No spinner — save is fast. |
| Save success | Card collapses to view mode with updated data. No toast. |
| Save error | Card stays in edit mode. Red `<Alert>` inside card: "Не удалось сохранить. Попробуйте ещё раз." |

### Settings Page

| State | Visual |
|-------|--------|
| Loading | Skeleton for input field. |
| Empty (no name set) | Input empty with placeholder "Как вас зовут?". |
| Error (save failed) | `<Alert variant="destructive">`: "Не удалось сохранить настройки." |
| Success (saved) | Brief green checkmark icon next to save button, fades after 2 seconds. |
