## Inline Edit UX

**Q: Клик на карточку — что визуально происходит?**
Options: Только description → textarea / Вся карточка разворачивается в форму / Клик редактирует только поле по которому кликнули
→ **Вся карточка разворачивается в форму** (карточка растёт вверх, все поля сразу видны)

**Q: Автосохранение сработало, но сеть пропала — что показывать?**
Options: Ошибка внутри карточки, поля остаются редактируемыми / Тост + поля закрываются (отмена правок)
→ **Ошибка внутри карточки, поля остаются редактируемыми** (правки не теряются)

**Q: Как визуально отличать карточку в режиме редактирования?**
Options: Рамка + кнопка отмены / Только поля превращаются в input / Световой фон карточки меняется
→ **Рамка + кнопка отмены** (голубая рамка вокруг карточки, крестик или «Отмена» в углу)

## Manager Actions

**Q: manager_actions — отдельная колонка или секция внутри Назначений?**
Пользователь: «Я предлагаю не отделять спикера от остальных действующих лиц. В транскрипции будут имена у всех действующих лиц, поэтому тут как будто дополнительно с точки зрения UI делать не надо, надо с точки зрения логики наполнения и продумывания задач.»
→ **Нет отдельной UI-секции** — задачи пользователя идут в assignments под его именем

**Q: Claude знает что делать руководителю — как?**

→ **Имя пользователя из настроек** — передаётся в промпт, Claude добавляет логически выведенные задачи для этого человека

**Q: Имя пользователя — где указывать?**
Options: Одноразово в настройках / Каждый раз на форме ввода
→ **Одноразово в настройках** (хранится в Supabase)
