# Phase 3.1: Record View Overhaul - Context

**Gathered:** 2026-03-31
**Status:** Ready for planning

<domain>
## Phase Boundary

Переработать record view с таб-навигации на канбан-layout. Обновить AI-схему и промпт (source_quote, ISO-дедлайны, пользовательское имя для инференса задач). Добавить поле meeting_date на форму ввода. Добавить hidden_blockers из interpretation в колонку Блокеры.

Не входит в фазу: история записей, генерация отчётов, drag-and-drop между колонками.

</domain>

<decisions>
## Implementation Decisions

### Канбан layout

- **D-01:** 5 колонок: Сделано / В работе / Блокеры / Назначения / Дедлайны — каждая с subtitle (пояснение под заголовком)
  - Сделано — *"что завершено"*
  - В работе — *"что сейчас делается"*
  - Блокеры — *"что мешает двигаться"*
  - Назначения — *"кто что делает"*
  - Дедлайны — *"ключевые даты"*
- **D-02:** В колонке Блокеры после явных блокеров отображаются `hidden_blockers` из interpretation — другой фон + бейдж "AI", клик раскрывает source (уже есть в InterpretationSchema через GroundedClaimSchema)

### Inline edit UX

- **D-03:** Клик на карточку → вся карточка разворачивается в форму (все редактируемые поля видны сразу, карточка растёт вверх). Не click-to-edit по конкретному полю.
- **D-04:** Клик вне карточки → автосохранение. Явной кнопки "Сохранить" нет.
- **D-05:** При ошибке автосохранения — ошибка показывается внутри карточки, поля остаются редактируемыми (правки не теряются).
- **D-06:** Карточка в режиме редактирования визуально отличается: рамка вокруг + кнопка отмены в углу. Отмена → возврат к исходным данным без сохранения.
- **D-07:** Редактируемые поля — все текстовые (description, person, deadline/by_when). Enums (impact, type) — read-only, как было в Phase 3.

### Deadline slot на карточках

- **D-08:** Slot с датой появляется только у карточек, где есть дата (`deadline` / `by_when`). Карточки без даты не имеют пустого слота.
- **D-09:** Дедлайн в slot отображается форматированной датой (не ISO строкой). Старые записи с текстовым deadline — показывать текст как есть (обратная совместимость не приоритет для MVP).

### Source quote

- **D-10:** Кнопка "источник" показывается только если у карточки есть `source_quote`. Если поля нет или оно null/пустое — кнопку не показывать.
- **D-11:** `source_quote` добавляется как опциональное поле (`source_quote?: string`) ко всем элементам ExtractedData: done, in_progress, blockers, assignments, deadlines.

### meeting_date

- **D-12:** Поле "Дата встречи" — date picker (не текстовое поле), открывает календарь. Дефолт = сегодня.
- **D-13:** Не блокирует сабмит — если не указана, используется текущая дата.
- **D-14:** Размещение на форме ввода: рядом с полем Label, над textarea, в одной строке.
- **D-15:** `meeting_date` передаётся в Claude как опорная дата для конвертации относительных упоминаний ("в пятницу", "до конца недели") в абсолютные ISO даты по московскому времени.

### Дедлайны как ISO дата

- **D-16:** Поле `date` в deadlines — ISO datetime строка (Москва, UTC+3), не свободный текст.
- **D-17:** Поле `deadline` в in_progress и `by_when` в assignments — тоже ISO datetime, если Claude может определить дату. Если дата не упоминается → null (не придумывать).

### Инференс задач пользователя (бывший manager_actions)

- **D-18:** Отдельного поля `manager_actions` в схеме нет. Claude добавляет логически выведенные задачи для пользователя прямо в массив `assignments` под именем пользователя.
- **D-19:** Имя пользователя хранится в настройках (один раз, не на каждой форме). Передаётся в промпт экстракции как `user_name`. Claude включает задачи для этого человека: follow-up по блокерам, проверка in_progress, уточнения по неясностям.
- **D-20:** Нет визуального разделения "мои задачи" vs "чужие" в UI — все назначения под именами участников в одной колонке.
- **D-21:** Нужна страница/секция настроек (`/settings` или inline на главной) для указания имени пользователя. Хранение — `user_settings` таблица в Supabase или localStorage (выбор за планировщиком, предпочтительно Supabase для консистентности).

### Claude's Discretion

- Точный визуальный стиль карточек (padding, тени, размер шрифта)
- Анимация раскрытия карточки в режиме редактирования
- Поведение скролла при переполнении колонки
- Точный формат отображения даты в deadline slot

</decisions>

<specifics>
## Specific Ideas

- Все карточки с `deadline`/`by_when` имеют отдельный слот внизу с датой и иконкой 📅

- Кнопка "источник" работает по тому же принципу, что и в interpretation tab (уже реализован EvidenceCollapsible)
- Канбан — это замена всей tab-навигации, не дополнение к ней
