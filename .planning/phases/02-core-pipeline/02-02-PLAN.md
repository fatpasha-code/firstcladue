1. Extend `src/app/actions.ts` -- keep existing `signOut`, add three new Server Actions per D-02 (staged loading requires separate actions so client can update status between calls):

   **`saveRecord(rawText: string, label?: string)`:**
   - Validate: `rawText` non-empty after trim, max 50000 chars (per D-04, INGEST-02)
   - Return `{ error: 'Вставьте текст для анализа' }` if empty
   - Return `{ error: 'Текст превышает лимит в 50 000 символов' }` if over limit
   - Get user via `supabase.auth.getUser()`, return `{ error: 'Не авторизован' }` if no user
   - Insert into `records` table: `{ user_id: user.id, raw_text: rawText, label: label || null, status: 'pending' }`
   - `.select('id').single()`
   - Return `{ id: record.id }` on success
   - try/catch: `console.error('[saveRecord]', error)`, return `{ error: 'Не удалось сохранить запись' }`

   **`extractRecord(recordId: string)`:**
   - Get user, verify auth
   - Fetch record by id where `user_id = user.id` (RLS enforces this too but belt-and-suspenders)
   - Update status to `'processing'`
   - Fetch `raw_text` from record
   - Call `runExtraction(record.raw_text)`
   - Update record: `{ extracted_data: extractedData }`
   - Return `{ success: true }`
   - On error: update record `{ status: 'failed', error_message: 'Ошибка извлечения данных' }`, return `{ error: 'Не удалось извлечь данные' }`

   **`interpretRecord(recordId: string)`:**
   - Get user, verify auth
   - Fetch record by id (must have `extracted_data`)
   - Call `runInterpretation(record.raw_text, record.extracted_data)`
   - Update record: `{ interpretation, status: 'completed', updated_at: new Date().toISOString() }`
   - Return `{ id: recordId, interpretation }` (client needs interpretation for inline display per D-01)
   - On error: update record `{ status: 'failed', error_message: 'Ошибка интерпретации' }`, return `{ error: 'Не удалось выполнить интерпретацию' }`

   Import types from `@/lib/ai/schemas` for proper typing of `interpretation` return value.

   **handleSubmit function (per D-02 staged loading with 3 separate actions):**
   ```typescript
   async function handleSubmit(e: React.FormEvent) {
     e.preventDefault()
     setError('')
     setResult(null)
     setIsSubmitting(true)

     try {
       // Step 1: Save
       setStage('Сохраняем запись...')
       const saveResult = await saveRecord(rawText, label || undefined)
       if ('error' in saveResult) { setError(saveResult.error); return }

       // Step 2: Extract
       setStage('Извлекаем данные...')
       const extractResult = await extractRecord(saveResult.id)
       if ('error' in extractResult) { setError(extractResult.error); return }

       // Step 3: Interpret
       setStage('Интерпретируем...')
       const interpretResult = await interpretRecord(saveResult.id)
       if ('error' in interpretResult) { setError(interpretResult.error); return }

       // Done
       setStage('Готово')
       setResult({ id: saveResult.id, interpretation: interpretResult.interpretation })
       setRawText('')
       setLabel('')
     } finally {
       setIsSubmitting(false)
       // Clear stage after brief delay so "Готово" is visible
       setTimeout(() => setStage(''), 1000)
     }
   }
   ```
