# UI Feedback — Phase 03 Review (2026-03-30)

Собранные правки по итогам использования record view. Основа для планирования следующей фазы.

---

## Правки

### A. Канбан-layout (замена табов)

- Вертикальный канбан: колонки расположены горизонтально, карточки внутри идут сверху вниз
- Клик на карточку → inline редактирование
- Клик вне карточки → автосохранение
- Все карточки, которые имеют ограничение по времени, должны иметь отдельный слот под дедлайн

### B. Дедлайны — конкретная дата вместо текста

- Дедлайн хранится как дата+время по Москве (ISO), не свободный текст
- Опорная дата для конвертации — **обязательное поле "Дата встречи"** на странице ввода
  - Дефолт: сегодня (покрывает большинство случаев)
  - Пользователь меняет вручную, если расшифровывает старый созвон
- Claude конвертирует все относительные упоминания ("сегодня", "в пятницу", "до конца недели") в абсолютную дату на основе `meeting_date`
- В UI — date picker
- Если дата задачи не упоминается, но по смыслу нужна проверка → попадает в manager_actions (пункт D) без даты

### C. Источник на каждой карточке

- Добавить `source_quote?: string` ко всем элементам `ExtractedData` (done, in_progress, blockers, assignments, deadlines)
- Claude при извлечении цитирует конкретный фрагмент текста — не весь транскрипт, а релевантный отрезок
- На каждой карточке — коллапсибл "источник" (аналог того, что уже есть в interpretation)
- Старые записи без source_quote — кнопка "источник" просто не показывается

### D. Действия руководителя в Назначениях

- Добавить в схему: `manager_actions: Array<{ action: string, context: string, source_quote?: string, deadline?: string }>`
- Claude заполняет НЕ только явные поручения, но и логически выведенные:
  - Есть задача in_progress → нужно получить апдейт / проверить выполнение
  - Есть блокер → нужно принять решение / разблокировать
- Отображается в колонке "Назначения" отдельной секцией (выделена визуально)
- Важно: показывать задачи ВСЕХ участников, не только подчинённых — ассистент, который не был на созвоне, должен видеть полную картину

### E. Пояснения под заголовками колонок

- Subtitle под каждым названием колонки:
  - Сделано — *"что завершено"*
  - В работе — *"что сейчас делается"*
  - Блокеры — *"что мешает двигаться"*
  - Назначения — *"кто что делает + мои действия"*
  - Дедлайны — *"ключевые даты"*

### F. Скрытые блокеры из интерпретации → в колонку Блокеры

- `hidden_blockers` из interpretation отображаются в колонке "Блокеры" после явных
- Визуально отличаются: другой фон + бейдж "AI"
- Клик раскрывает источник (уже есть в interpretation schema)
