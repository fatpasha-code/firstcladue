# PROJECT_IDEA.md — Update Tracker

**Текущий этап**: internal MVP
**Пользователь**: Павел (один человек)
**Статус**: не SaaS, не публичный продукт

---

## Проблема

Павел управляет разработчиком и тратит значительное время каждую неделю на:

- Разбор созвонов и переписок
- Попытку понять что сделано / что в работе / что заблокировано
- Перевод технических деталей в управленческий язык
- Ручное составление недельных отчётов

Результат:
- Отчёты часто неполные или составляются с задержкой
- Нет структурированного трекинга блокеров
- Время тратится на пересказ вместо анализа
- Технические апдейты разработчика трудно интерпретировать управленчески

---

## Решение

Инструмент, который:

1. Принимает текст (созвон / переписка / заметки)
2. Выделяет структурированные данные:
   - что сделано
   - что в работе (кто, дедлайн)
   - блокеры (явные и неявные)
   - кому что назначено
   - дедлайны (явные и скрытые)
3. Генерирует:
   - недельный digest
   - месячный отчёт
   - список follow-up
4. Делает управленческую интерпретацию технических апдейтов:
   - объясняет что происходит на самом деле
   - выявляет скрытые блокеры и риски
   - указывает неясности и что нужно уточнить

---

## Целевой результат для MVP

Для внутреннего использования:
- Копировать текст из созвона / Slack → вставить → получить отчёт
- Тратить на еженедельный отчёт минуты, а не часы
- Понимать технические апдейты разработчика без необходимости переспрашивать

---

## Архитектура

```
Вставить текст
     ↓
Extraction (Claude API)
  → что сделано, блокеры, дедлайны, назначения
     ↓
Interpretation (Claude API)
  → управленческий разбор, неясности, что уточнить
     ↓
Сохранение в БД (Supabase)
     ↓
History & Review
  → просмотр, редактирование, подтверждение
     ↓
Reports
  → недельный / месячный / follow-up
```

---

## Стек (v1)

- Next.js (App Router) + Tailwind CSS + shadcn/ui
- Supabase (PostgreSQL + Auth)
- Vercel (deploy)
- Anthropic SDK (Claude API)

Не в v1: платёжки, product analytics, observability инструменты, team access.

---

## Метрики успеха (internal MVP)

- [ ] Отчёт генерируется за < 2 минут после вставки текста
- [ ] Extraction полезна после ручного просмотра (видны нужные блоки)
- [ ] Управленческая интерпретация понятна и полезна
- [ ] Инструмент используется регулярно самим Павлом
- [ ] История анализов сохраняется и доступна

---

## Риски

| Риск | Вероятность | Mitigation |
|------|-------------|-----------|
| Extraction ошибается | Средняя | UI для ручного редактирования results |
| API тормозит / падает | Средняя | Retry с backoff, graceful error message |
| Стоимость Claude API | Средняя | Оптимизация промптов, оценить реальный расход |
| Сложность для одного пользователя | Низкая | MVP без лишних фич |
